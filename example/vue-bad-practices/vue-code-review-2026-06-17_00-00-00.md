## Issues Found

1. **Options API used instead of `<script setup>` Composition API**
    - **Severity**: High.
    - **Description**: Both `Todo.vue` and `TodoItem.vue` use the Options API with `defineComponent` and `export default`. The guidelines mandate `<script setup>` syntax with the Composition API. Options API components are harder to tree-shake, produce worse TypeScript inference, and accumulate `this`-binding bugs as components grow.
    - **Location**: [src/components/Todo.vue:24-57](src/components/Todo.vue#L24-L57), [src/components/TodoItem.vue:8-28](src/components/TodoItem.vue#L8-L28).
    - **Recommendation**: Rewrite both components using `<script setup lang="ts">`. Replace `data()` with `ref`/`reactive`, `methods` with plain functions, and `emits` with `defineEmits<{ ... }>()`.

2. **Index used as `key` in `v-for`**
    - **Severity**: High.
    - **Description**: `:key="i"` uses the loop index as the key for `TodoItem`. When items are reordered, added in the middle, or removed, Vue cannot distinguish between nodes — it patches the wrong element, causing stale DOM state, broken animations, and subtle component lifecycle bugs (e.g., a checked item appearing unchecked after deletion of a sibling above it).
    - **Location**: [src/components/Todo.vue:15](src/components/Todo.vue#L15).
    - **Recommendation**: Add a stable `id` field to each todo item (e.g., `crypto.randomUUID()` or a monotonic counter) and bind `:key="t.id"` instead.

3. **Direct mutation of a prop object (`flipIt` mutates `x.done`)**
    - **Severity**: High.
    - **Description**: `flipIt(x)` receives the todo object reference and writes `x.done = !x.done` directly. This mutates an object owned by the parent through its reference — a prop mutation anti-pattern. Even though it happens to work here because object references flow downward, it breaks the unidirectional data flow contract, makes change tracking impossible, and will silently break once the data source is lifted into a store or the parent uses `shallowRef`.
    - **Location**: [src/components/Todo.vue:49-51](src/components/Todo.vue#L49-L51).
    - **Recommendation**: Emit the item identity and let the parent own the mutation. Pass an `id` as the event payload from `TodoItem` (`emit('flip', props.todo.id)`), then let `Todo.vue` find and toggle the item in its own reactive array.

4. **Non-descriptive variable, prop, and method names throughout**
    - **Severity**: Medium.
    - **Description**: Multiple identifiers carry no semantic meaning: `n` (new text input), `stuff` (todo list), `t` (todo), `i` (index), `v` (trimmed value), `x` (prop name in `TodoItem`), `y` (filter comparand), `a` and `b` (toggle and delete handlers). This makes the code nearly unreadable at a glance and will slow down any future contributor.
    - **Location**: [src/components/Todo.vue:35-54](src/components/Todo.vue#L35-L54), [src/components/TodoItem.vue:14-27](src/components/TodoItem.vue#L14-L27).
    - **Recommendation**: Rename to meaningful identifiers: `newTodo`, `todos`, `todo`, `idx`, `trimmedValue`, `todo` (prop), `handleToggle`, `handleDelete`. Event handlers should use the `handle` prefix per the guidelines.

5. **Component name not PascalCase in `TodoItem.vue`**
    - **Severity**: Medium.
    - **Description**: `name: 'todoitem'` uses all-lowercase. Vue DevTools and error messages will display `todoitem` instead of `TodoItem`, making debugging harder. The guidelines explicitly require PascalCase for component names.
    - **Location**: [src/components/TodoItem.vue:12](src/components/TodoItem.vue#L12).
    - **Recommendation**: Change `name: 'todoitem'` to `name: 'TodoItem'`. When migrating to `<script setup>`, the filename-derived name is used automatically — no explicit `name` needed.

6. **Emits not typed with a dedicated interface**
    - **Severity**: Medium.
    - **Description**: `emits: ['flip', 'erase']` is declared as a plain string array. This provides no payload type information to consumers or to TypeScript. The guidelines require using `defineEmits` with a typed interface so callers know what shape the event payload carries.
    - **Location**: [src/components/TodoItem.vue:19](src/components/TodoItem.vue#L19).
    - **Recommendation**: After migrating to `<script setup>`, use:
      ```ts
      const emit = defineEmits<{
        flip: [todo: Todo]
        erase: [todo: Todo]
      }>()
      ```

7. **No shared type definition for the Todo model**
    - **Severity**: Medium.
    - **Description**: `{ text: string; done: boolean }` is duplicated as an inline type in `Todo.vue` (methods) and `TodoItem.vue` (prop definition). If the shape ever grows (e.g., adding `id`, `createdAt`), both files must be updated separately, and any missed location is a silent type gap.
    - **Location**: [src/components/Todo.vue:49,52](src/components/Todo.vue#L49), [src/components/TodoItem.vue:15](src/components/TodoItem.vue#L15).
    - **Recommendation**: Extract a shared type to `src/types/todo.type.ts`:
      ```ts
      export interface Todo {
        id: string
        text: string
        done: boolean
      }
      ```
      Import and reuse it in both components.

8. **Complex ternary expression rendered directly in the template**
    - **Severity**: Low.
    - **Description**: `{{ x.done ? '✓ ' : '' }}{{ x.text }}` mixes conditional icon logic inline. The guidelines prefer moving non-trivial logic to computed properties or helper expressions outside the template to keep templates declarative and readable.
    - **Location**: [src/components/TodoItem.vue:3](src/components/TodoItem.vue#L3).
    - **Recommendation**: After migrating to `<script setup>`, expose a computed `label`:
      ```ts
      const label = computed(() => (props.todo.done ? '✓ ' : '') + props.todo.text)
      ```
      Then simplify the template to `{{ label }}`.
